# TeamSpeak 3 в Docker

Запуск сервера TeamSpeak 3 через Docker Compose. Все данные хранятся в каталоге `teamspeak-data` — его можно копировать на другой сервер для переноса.

## Требования

- Docker и Docker Compose
- Порты: **9987/UDP** (голос), **30033/TCP** (файлы). ServerQuery отключён — для голоса и файлов не нужен.

## Быстрый старт

1. Скопируйте пример настроек и при необходимости отредактируйте порты:
   ```bash
   cp .env.example .env
   ```

2. Запустите контейнер:
   ```bash
   docker compose up -d
   ```

3. При первом запуске в логах появится **privilege key** (ключ администратора). Сохраните его:
   ```bash
   docker compose logs teamspeak
   ```
   Ищите строку `token=...` — это ключ для получения прав админа в клиенте (Права → Использовать ключ привилегий).

4. Подключитесь клиентом TeamSpeak к адресу сервера (порт по умолчанию 9987/UDP). Для получения прав администратора используйте privilege key (Права → Использовать ключ привилегий).

## Порты и файрвол

Откройте на хосте (или пробросьте с роутера):

| Порт   | Протокол | Назначение       |
|--------|----------|------------------|
| 9987   | UDP      | Голосовой трафик |
| 30033  | TCP      | Передача файлов  |

## Данные и перенос на другой сервер

Вся конфигурация и данные сервера лежат в каталоге **`teamspeak-data`** (монтируется в `/var/ts3server`):

- база данных;
- логи;
- `licensekey.dat` (если используете лицензию);
- настройки виртуального сервера.

Чтобы перенести сервер на другой хост:

1. Остановите контейнер: `docker compose down`
2. Скопируйте каталог `teamspeak-data` и файлы `docker-compose.yml`, `.env` на новый хост
3. На новом хосте: `docker compose up -d`

Регулярно делайте резервные копии `teamspeak-data`.

## Лицензия

- Для принятия лицензионного соглашения в `.env` должно быть `TS3SERVER_LICENSE=accept`.
- Файл лицензии `licensekey.dat` (для более чем 32 слотов и др.) положите в `teamspeak-data/` и перезапустите контейнер. Путь к каталогу с ключом можно задать через `TS3SERVER_LICENSEPATH` в `.env`.

## Полезные команды

```bash
docker compose up -d      # Запуск
docker compose down       # Остановка
docker compose logs -f    # Логи в реальном времени
docker compose ps         # Статус контейнера
```
